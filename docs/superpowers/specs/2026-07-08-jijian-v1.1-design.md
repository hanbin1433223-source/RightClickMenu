# 即建 v1.1 设计文档

## 设计目标

- 解决 UI 过于扁平的问题，采用 macOS 27 Liquid Glass 设计语言
- 引入 NSPopover 代替纯文字菜单
- 首次引导 → 类型管理 → 快捷键配置 → 开机自启

## 架构变更

当前：`@main → MenuBarExtra(.menu) → 纯文字列表`
改进：`@main → NSStatusItem + NSPopover → PopoverView(SwiftUI)`

## 文件变更

### 新增
- `PopoverView.swift` — NSPopover 内容视图
- `Settings.swift` — 配置模型 + 文件类型定义
- `OnboardingView.swift` — 首次引导向导

### 修改
- `App.swift` — 替换 MenuBarExtra 为 NSStatusItem + NSPopover
- `ContentView.swift` — 三标签页设置窗口
- `rclick/main.swift` — 增加 13 种新类型
- `Package.swift` — 新增 Carbon linker flag

### 删除
- `docs/design/jijian-style-preview.html` (已删)
- `generate_icon.swift` (已删)

## NSPopover 布局

宽 280pt，高自动适配内容。

```
┌─────────────────────┐
│ 📄 即建             │ ← 标题
├─────────────────────┤
│ 📝 文本文档          │
│ 📄 Markdown          │ ← 已启用的文件类型列表
│ 📄 富文本文档        │   每个类型配 SF Symbol
│ 📄 Word 文档         │
│ 📄 Excel 表格        │
│ 📄 PowerPoint        │
├─────────────────────┤
│ ⚙️ 设置...           │ ← 打开设置窗口
│ ⏻ 退出               │
└─────────────────────┘
```

## 设置窗口 — 三标签页

标签页 1 — 使用说明
标签页 2 — 文件类型开关（按分类分组）
标签页 3 — 高级（快捷键选择、快速创建模式、默认类型、开机自启）

## 首次引导 — 两步向导

Step 1: 选择使用模式（快捷键优先 / 菜单栏优先 / 两者都要）
Step 2: 选择文件类型（检测已安装软件）

## 配置模型

UserDefaults 存储 JSON 序列化的 AppSettings。

## 技术栈

- NSStatusItem + NSPopover（AppKit）
- PopoverView（SwiftUI，NSHostingController 桥接）
- Carbon RegisterEventHotKey（快捷键）
- SMAppService（开机自启）
