# 即建 v1 产品规格

## 项目定位

即建是一款 macOS Finder 右键增强工具，解决 Mac 新建文本、Markdown、Office 文档不方便的问题。

项目第一版目标是发布到 GitHub，用户下载源码后可以用 Xcode 构建并使用。第一版不追求商业化发布，不做签名公证和安装包，优先打通完整可用链路。

项目管理原则：第一版先稳定可用，第二版增强体验，第三版产品化。新增需求默认先判断属于 v1、v2 还是 v3，避免第一版失控。

## 品牌与视觉

- App 名称：即建
- Finder 右键菜单名称：即建
- GitHub 仓库/工程建议名：RightClickMenu
- Slogan：在 Finder 右键，即刻新建文件
- 语言：中文优先
- 风格关键词：原生感、轻量、可靠、中文友好
- 图标方向：文档 + 加号
- 颜色方向：浅灰白背景，青蓝色主强调，少量绿色辅助
- UI 基调：像 macOS 原生工具，清爽可信，不做花哨营销风
- 视觉预览：`docs/design/jijian-style-preview.html`

## 第一版范围

第一版必须完成：

- SwiftUI 主 App，展示使用说明和文件类型开关。
- FinderSync Extension，在 Finder 右键菜单中显示“即建”二级菜单。
- 支持创建 `txt`、`md`、`rtf`、`docx`、`xlsx`、`pptx`。
- 自动命名文件，例如 `新建文本文档.txt`。
- 重名时自动追加编号，例如 `新建文本文档 2.txt`。
- 默认监听用户主目录，覆盖桌面、文稿、下载等常用目录。
- 主 App 和 Extension 通过 App Groups 同步配置。
- README 提供面向新手的详细构建、启用、使用说明。
- 提供 MIT License。
- 提供基础 App 图标。

第一版不做：

- App Store 发布
- 签名公证
- 安装包
- 自动更新
- 登录或云同步
- 付费功能
- 用户自定义文件类型
- 复杂模板编辑器
- 多语言国际化
- 菜单栏工具
- Finder 私有 API 或注入 Finder

## 第一版开发板块

### 工程骨架

创建完整 Xcode 工程结构，包括主 App target、FinderSync Extension target、基础 `Info.plist`、`Assets.xcassets` 和 `generate_pbxproj.py`。

成功标准：Xcode 能打开工程，主 App 和 Extension target 能被识别并构建。

### 文件创建核心

实现文件类型模型、文件创建服务和模板管理。创建文件时不能覆盖已有文件，必须自动生成唯一文件名。

成功标准：给定目录和文件类型，可以创建正确文件，并且重名时不会覆盖旧文件。

### Finder 右键菜单

实现 FinderSync Extension，注册监听目录，构建“即建”二级菜单，并调用文件创建逻辑。

成功标准：用户在 Finder 右键菜单中能看到“即建”，并能创建文件。

### 主 App 设置界面

实现新手友好的 SwiftUI 界面，包括使用步骤、扩展启用提示、文件类型开关。

成功标准：用户打开 App 后知道它是什么、如何启用、如何使用。

### GitHub 发布资料

准备 README、LICENSE、开发规范和常见问题说明。

成功标准：新手用户看 README 可以完成源码下载、Xcode 构建、扩展启用和基础使用。

### 图标与基础视觉

根据已确认风格制作基础图标和 App 内视觉样式。

成功标准：App 具有基本成品感，名称、图标、界面风格统一。

### 测试与验收

验证构建、扩展启用、右键菜单、文件创建、重名处理、配置同步和 README 步骤。

成功标准：第一版可以作为 GitHub 开源项目发布。

## 并行开发安排

可以拆成多个对话框并行推进：

- 对话框 A：工程骨架
- 对话框 B：文件创建核心
- 对话框 C：FinderSync Extension
- 对话框 D：主 App 设置界面
- 对话框 E：README 和发布资料
- 对话框 F：图标和视觉资源
- 对话框 G：总集成和验收

并行规则：

- 工程骨架先完成。
- 文件创建核心和主 App 设置界面可以并行。
- FinderSync Extension 依赖工程骨架和文件创建接口。
- 总集成必须最后进行。
- 每个对话框只负责自己的板块，避免同时修改同一核心文件。

## 第二版计划

第二版目标是让即建从“能用”变成“更好用”。

建议功能：

- 创建完成后自动选中新文件，方便用户重命名。
- 支持用户自定义文件类型。
- 支持编辑默认文件内容。
- 支持调整菜单显示顺序。
- 增加更友好的错误提示。
- 增加首次启动引导页。
- 增加扩展未启用检测和提示。
- 增加截图版使用教程。
- 优化 UI 和图标细节。

## 第三版计划

第三版目标是让即建从“开源小工具”变成“长期维护产品”。

建议功能：

- 模板库，支持用户导入自己的 Word、Excel、PPT 模板。
- 模板导入导出。
- 按目录显示不同文件类型菜单。
- 菜单栏入口。
- 快捷键入口。
- 自动更新。
- 签名、公证、打包下载。
- 英文 README 和国际化。
- GitHub Pages 或官网介绍页。

## 验收清单

- Xcode 能打开工程。
- 主 App 能运行。
- FinderSync Extension 能构建。
- 系统设置里能看到 Finder 扩展。
- 启用扩展后 Finder 右键能看到“即建”。
- 六种文件都能创建。
- 重名不会覆盖已有文件。
- 关闭某种文件类型后右键菜单同步隐藏。
- README 步骤可以被新手按顺序完成。
- App 名称、图标、UI、文档统一使用已确认品牌基调。
